## What is Hy3 Research Studio?

Hy3 Research Studio is an **AI-native research and creation workbench** powered by Tencent's Hy3 large language model. Unlike generic chatbots, it's designed around the complete research workflow: from a vague idea, through multi-source literature retrieval and evidence compression, to hypothesis generation, gap identification, experiment design, and a fully cited academic report. On top of that, it introduces the concept of **AI Native Features** — users don't just write prompts; they create standalone micro-apps with custom layouts, system prompts, and guided interactions — all generated by Hy3 in 30 seconds from a single sentence.

**No Node.js, no database, no Docker required.** Just Python 3.10+ and a Hy3 API key.

## 🚀 Quick Start

### Prerequisites

- Python 3.10 or higher
- A modern browser (Chrome / Edge / Firefox)
- A Hy3 API key (from Tencent TokenHub, or any OpenAI-compatible endpoint)

### 3 Steps to Run

```bash
# 1. Clone the repository
git clone https://github.com/Xtdzs/Hy3-Research-Studio.git
cd "Hy3 Research Studio"

# 2. Install dependencies (7 packages, no Node.js needed)
pip install -r requirements.txt

# 3. Configure your API key
# Windows PowerShell:
Copy-Item .env.example .env; notepad .env
# macOS/Linux:
# cp .env.example .env && nano .env
```

Edit `.env` and fill in your API key:

```env
HY3_API_KEY=sk-your-key-here
HY3_BASE_URL=https://tokenhub.tencentmaas.com/v1
HY3_MODEL=hy3
```

Then start the server:

```bash
python run.py
```

Open **http://localhost:8731** in your browser. That's it!

> All data (research tasks, library, custom features, settings) is persisted as JSON files in the `data/` directory. No external database needed.

## 🧩 Feature Modules

### 1. 🔬 Deep Research (深度研究) — Flagship

The core of the studio. Input a research topic, and Hy3 orchestrates an **8-stage automated research pipeline**:

1. **Planning** — Hy3 decomposes the topic into 2-5 sub-questions, generates English search queries for each, and drafts a report outline
2. **Multi-source Retrieval** — Parallel searches across OpenAlex, Crossref, and arXiv; keyword filtering + LLM semantic relevance judging
3. **Evidence Compression** — Instead of dumping raw chunks into context, Hy3 compresses literature abstracts into structured **Evidence Packets** (claim / method / limitation / supporting citations), achieving 5:1~10:1 compression
4. **Hypothesis Generation** — Derives 3-5 falsifiable research hypotheses with confidence scores and supporting evidence
5. **Evidence Graph** — Identifies logical relationships between evidence packets (support / contradict / extend / specialize)
6. **Research Gaps** — Mines uncovered directions from evidence limitations
7. **Experiment Design** — Proposes validation experiments (method / dataset / metrics / baselines / expected results)
8. **Report Writing** — Section-by-section streaming generation with inline `[sN]` citations; click any citation to view the source paper in a side panel

**Key features:**
- Real-time SSE progress bar showing all 8 stages
- Streaming output for report sections (typewriter effect)
- Clickable `[sN]` citations with source paper previews
- Section-level refinement: expand, condense, change tone, add citations, write counter-arguments
- Tabs for Evidence Packets, Hypotheses, Evidence Graph, Gaps, and Experiments
- Configurable depth (Quick / Standard / Deep), target citation count, style (academic / technical / popular), and language

### 2. 🔍 Smart Search (智能检索)

Multi-source academic search with AI-powered assistance:
- Searches across **OpenAlex, Crossref, arXiv** simultaneously (Semantic Scholar optional with API key)
- Hy3 automatically optimizes search queries (Chinese→English rewriting for academic indexes)
- LLM semantic filtering removes irrelevant results
- Streaming generation of a structured search brief with citations
- One-click save papers to Library
- Filters: year range, paper type, open access only

### 3. 📄 Paper Seminar (论文研讨)

Upload a PDF and chat with the paper:
- PDF text extraction via pypdf
- Ask questions about methodology, contributions, results, limitations
- Auto-extract outline, key innovations, and experimental results
- Multi-turn conversation with full paper context
- Upload history is saved for later review

### 4. ✍️ Writing Studio (写作工坊)

Dedicated writing assistance tools:
- **Abstract Generator** — Write concise abstracts from key points
- **Outline Generator** — Structure papers and proposals
- **Paragraph Writer** — Expand bullet points into full paragraphs
- **Review Writer** — Draft literature reviews with citations
- Streaming output for all writing tasks
- History of previous writing sessions

### 5. 💡 Idea Refiner (思路提炼) — Agent Mode

An AI research coach that uses **Agent + Tool Calling** to help users sharpen research directions:
- **Forced retrieval before answering**: Hy3 must call the `retrieve_papers` tool before responding, ensuring all claims are grounded in real literature
- **Smart retrieval decisions**: Simple greetings skip retrieval; research questions trigger automatic searches
- **Chinese query optimization**: Detects Chinese queries, rewrites to 2-5 English keywords, retries if no results
- **Two-layer relevance filtering**: Keyword rules first, then LLM semantic judgment — irrelevant papers are discarded before reaching context
- Answers include `[rN]` citations; click to see source details
- **Convergence mechanism**: After 3+ rounds, Hy3 guides the conversation toward an actionable research plan
- **Follow-up choices**: After each answer, Hy3 suggests 2-4 specific next steps
- **Generate Guide**: Converges multi-turn discussion into a structured research framework document

### 6. 🔨 Feature Workshop (创造工坊) — Flagship

**AI Native Feature Platform** — users don't just save prompts; they create standalone AI micro-apps with dedicated interfaces.

**Two creation modes:**
- **⚡ AI Quick Build** — Describe what you want in one sentence; Hy3 generates a complete feature in 30 seconds (name, emoji, description, system prompt, layout type, starter prompts). Zero learning curve.
- **🎨 Visual Builder** — Multi-turn conversation to clarify requirements → Hy3 generates a draggable component layout → preview → save. For designers, PMs, and researchers.

**Dedicated Workspace Architecture:**
- Each feature gets its own URL (`#/feature/{id}`) and independent workspace
- Layouts are specialized per use case (not generic chat bubbles):
  - 📚 **Research Workspace** — 3-column: literature library + reading area + notes & citations
  - 🔍 **Review Workspace** — Review-dedicated: scoring dimensions + review output
  - 🎙️ **Meeting Assistant** — 2-column: transcript input + minutes & action items
  - 🏥 **Medical Report** — 2-column: patient info form + medical record output
- Stable exit/re-entry navigation (hash routing with proper state reset)

**Community ecosystem:**
- Star rating system (1-5 stars, weighted average)
- Fork / Remix — build on top of others' features
- Usage statistics per feature
- Category browsing (10 categories: research, writing, analysis, teaching, medical, legal, business, daily, creative, other)
- Favorites collection

### 7. 📚 Library (文献库)

Personal literature management:
- Save papers from search results across modules
- Organize with folders
- View paper details (title, authors, year, abstract, source)
- Add personal notes
- Cross-session persistence via JSON storage

### 8. 📊 Feedback Board (反馈看板)

Community feedback and issue tracking:
- Submit feedback with categories
- Word cloud visualization of common topics
- Upvote/downvote feedback items
- Priority sorting by votes

### 9. 🕐 History (历史记录)

Unified cross-module activity history:
- Track research tasks, searches, paper discussions, feature usage
- Resume previous sessions
- Filter by module type

### 10. 👤 Profile (个人主页)

Personalization settings:
- Identity and affiliation
- Research interests (used for personalized recommendations)
- Two-tier personalization: immediate signals (recent searches/topics) + stable profile (declared interests)
- "Guess you want to search" suggestions based on combined signals

### 11. ⚙️ Settings (设置)

System configuration:
- API key and model settings
- Retrieval source toggles (OpenAlex / Crossref / arXiv / Semantic Scholar)
- Default research depth and style preferences
- Model connection status indicators
- Data management (clear history, reset)

## 🧠 Hy3's Roles in the System

Hy3 serves as the reasoning engine across all modules, taking on 16 distinct roles:

| Role | Module | What it does |
|------|--------|-------------|
| Research Planner | Deep Research | Rewrites goals, decomposes sub-questions, generates search queries, designs report outline |
| Search Query Optimizer | Search / Advisor | Chinese→English query rewriting, core keyword extraction |
| Evidence Compressor | Deep Research | Compresses raw abstracts into structured evidence packets (5:1+ ratio) |
| Hypothesis Generator | Deep Research | Derives falsifiable hypotheses with confidence scores |
| Evidence Graph Builder | Deep Research | Identifies support/contradict/extend/specialize relationships |
| Gap Finder | Deep Research | Mines uncovered research directions from evidence limitations |
| Experiment Designer | Deep Research | Designs validation experiments (method/data/metrics/baselines) |
| Report Writer | Deep Research / Smart Search | Section-by-section streaming long-form report generation |
| Refiner | Deep Research | Section expansion/condensation/style change/citation addition/counter-argument |
| Relevance Judge | Search Modules | LLM semantic filtering: judges paper-by-paper relevance to the query |
| Research Coach | Idea Refiner | Multi-turn research direction guidance with forced retrieval and citations |
| Paper Analyst | Paper Seminar | Answers paper questions, extracts outline/innovations/results |
| Feature Generator | Feature Workshop | One-sentence requirement → complete AI micro-app |
| Feature Designer | Feature Workshop | Multi-turn requirement understanding for component layout generation |
| Suggestion Engine | Home / Search | Personalized search suggestions from two-tier user profile |
| LLM-as-Judge | Evaluation | Assesses evidence coverage and key point recall |

## 📊 Evaluation Framework

The project includes a runnable evaluation script comparing three systems:

| System | Description |
|--------|-------------|
| `baseline_A` | Direct Chat: one-shot model response without any retrieval |
| `baseline_B` | Search + Single-pass: retrieval then dump all abstracts into one prompt (no compression/alignment) |
| `studio` | Full Hy3 Research Studio 8-stage pipeline |

```bash
python -m tests.evaluate --quick    # Quick eval (2 tasks)
python -m tests.evaluate            # Full eval
python -m tests.test_offline        # Offline unit tests (no API key needed)
```

Metrics:
- **Efficiency**: Total token consumption, end-to-end latency, TTFP/TTFE/TTFR/TTR timings
- **Compression**: Evidence Compression Ratio (before/after character ratio)
- **Quality**: Evidence Coverage (LLM-as-judge key point recall rate)

## 📝 Notes

- This project calls Hy3 entirely through API; there is **no training, fine-tuning, or local model deployment**
- Search relies on OpenAlex / Crossref / arXiv public APIs; if unavailable, the UI remains browsable and features have offline fallbacks
- All data is stored as JSON in the `data/` directory; backup or delete to reset
- When `HY3_API_KEY` is not configured, the system enters offline mode: feature workshop uses heuristic layout inference, other AI features prompt for key configuration
